## 3. 架构

### 后端 (Rust → smartdns_plus_ui.so)

- **`src/config_ui.rs`** — 核心配置逻辑
  - `BeginnerConfig` — 基础设置 + upstreams + extras (HashMap 存任意官方指令)
  - `ManagedAssets` — 托管规则资产 (rule_groups, client_rules, ip_rules, routing_items, domain_sets, ip_sets)
  - `SaveConfigRequest` — 保存请求体
  - `save_beginner_config()` — 保存到 `smartdns-working.conf`（工作文件，不直接覆盖 smartdns.conf）
  - `apply_working_config()` — 将工作文件覆盖到 `smartdns.conf` + 备份
  - `parse_managed_assets_from_text()` — 解析托管规则文本
  - `render_managed_assets_text()` — 序列化托管规则
  - `render_basic_config()` — 生成基础配置文本（按缓存/测速/日志分段）
  - `render_upstreams_config()` — 生成上游 DNS 配置（按分组排列）

- **`src/http_server_api.rs`** — API 路由
  - `POST /api/plus/save` — 保存到工作文件
  - `POST /api/plus/save-and-restart` — 保存 + 重启
  - `POST /api/plus/apply` — 应用工作文件（覆盖 smartdns.conf + 重启）
  - `GET /api/plus/managed/assets` / `POST` — 托管资产读写
  - 其余为标准 SmartDNS WebUI API

### 前端 (wwwroot/)

- **`index.html`** — 单页应用，11 页导航 + 登录浮层 + 上游 DNS 模态框
- **`app.js`** — 所有逻辑 (~940 行)
- **`style.css`** — 所有样式

### 数据流

```
用户操作 → 修改 S.managed / S.upstreams 内存状态
         → autoSave() 500ms 后调用 POST /api/plus/save → 写入 smartdns-working.conf
用户点击"保存并重启" → POST /api/plus/apply → 覆盖 smartdns.conf → 重启容器
```

## 4. 已完成的功能

### 页面导航（11 页）
1. 🖥 基础设置 — 监听、运行参数、代理、Hosts、CA 证书、conf-file
2. 🌐 上游 DNS — 6 协议 × 15 选项，按分组展示，卡片编辑/删除，模态框新增
3. ⚡ 缓存与性能 — cache-size、TTL、过期缓存、双栈、查询控制
4. 🔀 上游分流 — nameserver，可视化表单（域名集合/后缀/域名 + 上游组下拉），拖拽排序
5. 📋 规则 — domain-rules（多选项）、address/cname、IP 规则
6. 📁 集合管理 — domain-set + ip-set 统一列表
7. 👥 规则组 — group-begin/end + group-match
8. 💻 客户端规则 — IP/CIDR/MAC → 规则组
9. 📋 日志与审计 — log-* / audit-*
10. 🔧 高级网络 — force-qtype-SOA、ECS、DNS64、DDNS
11. 📄 配置预览 — 实时显示 smartdns-working.conf

### 后端特性
- `extras: HashMap<String, String>` — 任意官方指令自动解析/渲染，无需改 Rust 代码
- 保存到工作文件，应用时才覆盖主配置
- 插件指令自动过滤（smartdns-ui.* 不写入输出）
- 配置输出按段落组织（缓存、测速、日志等）

## 5. 已知问题

### 🔴 严重：删除按钮无响应
- 上游分流和规则页面的删除按钮点击无效
- 已添加 `window.onerror` 全局错误提示，会弹窗显示 JS 错误
- 怀疑 `card.querySelector(".rt-del-btn")` 返回 null（HTML 模板问题）
- 修复方向：检查 renderRouting 和 renderDomainRules 中按钮的 DOM 结构

### 🟡 上游分流编辑功能
- 编辑按钮展开的表单引用了旧的类型下拉（RT_LABEL），但上游分流只有 nameserver

### 🟡 规则页删除
- renderDomainRules 的删除操作 items.splice，但 items 是局部变量，未正确同步到 S.managed

### 🟡 自动保存延迟
- autoSave 500ms 防抖，刷新太快可能丢失

### 🟢 可优化
- 域名规则/地址改写新增后直接 append DOM，未调用 render 函数刷新全部列表
- 上游分流删除后 S.managed.routing_items 重建逻辑需要验证
